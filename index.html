How It Works

1

Upload CSV

Drag and drop your geographic dataset. The wizard detects columns automatically.

2

Configure

Set column types, choose filters, pick tooltip and modal fields, style the map.

3

Export

The wizard generates a config file and SQL scripts ready for Supabase.

4

Deploy

Run the SQL in Supabase, drop the app folder into a repo, enable GitHub Pages. Done.

Features

Config-Driven

A single JSON file controls filters, tooltips, modals, statistics, map style — everything.

Interactive Map

Leaflet-powered with marker clustering, measurement tools, and multiple base layers.

Sidebar Filters

Dynamic dropdowns generated from your data. Filter combinations update the map in real time.

Global Search

Full-text search across configurable columns with instant results.

Query Builder

Advanced filtering with custom conditions — no SQL knowledge needed for end users.

Zero Build Step

Pure HTML, CSS, and ES6 modules. No bundler, no framework, no dependencies to install.

What You Get

webgis.config.json

Your entire site configuration: project info, Supabase credentials, column definitions, filters, display settings, map options.

setup.sql

Creates the database table, indexes, and row-level security policies in Supabase.

data.sql

INSERT statements for all rows in your CSV, ready to run in the Supabase SQL Editor.
